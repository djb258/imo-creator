## 1. Definitions

**Garage**: The imo-creator repository. Sovereign authority (CC-01). All governance originates here.

**Car**: A child repository that conforms to imo-creator doctrine. Cars receive governance. Cars do not define governance.

**Certification**: A cryptographically signed artifact that attests a work packet was executed within doctrine boundaries, audited against deterministic rules, and found compliant.

**Work Packet**: The execution contract between human intent and agent execution. Generated by Planner, executed by Builder (formerly Worker), evaluated by Auditor, certified by Signature Engine.

---

## 2. Governance Centralization

All governance infrastructure resides in the Garage. This is not a preference. This is structural law.

### Garage-Exclusive Artifacts

The following artifacts exist in imo-creator only. Child repos must not contain them:

| Artifact | Location | Purpose |
|----------|----------|---------|
| Agent definitions | `factory/agents/agent-*/` | Planner, Builder, Auditor, DB Agent skill prompts |
| WORK_PACKET schema | `factory/contracts/work_packet.schema.json` | V2 execution contract |
| Certification schema | `factory/contracts/certification.schema.json` | Merge-blocking signed artifact |
| Doctrine registry | `law/registry/doctrine_registry.json` | Machine-readable doctrine inventory |
| Taxonomy registry | `law/registry/taxonomy_registry.json` | Classification enums |
| Audit rules | `law/registry/audit_rules.json` | Deterministic evaluation rules |
| Execution contracts | `factory/runtime/` | Mount, runner, writer contracts |
| Certification infrastructure | `sys/certification/` | Signature engine, validator |
| Orchestration contract | `app/garage/orchestration_contract.json` | Pipeline definition |

### Car-Permitted Artifacts

Child repos may contain:

| Artifact | Purpose |
|----------|---------|
| `.garage/certification.json` | Written by Garage artifact_writer. Read-only in child repo. |
| `.garage/work_packet.json` | Copy of executed work packet. Read-only in child repo. |
| `.garage/audit_report.json` | Audit classification. Read-only in child repo. |
| `DOCTRINE.md` | Conformance declaration (copied from template). |
| `IMO_CONTROL.json` | Structural governance contract (copied from template). |
| `REGISTRY.yaml` | Hub identity and sub-hub declarations. |

Child repos do not own `.garage/` contents. The Garage writes. The child repo reads.

## 3. Boundary Enforcement

### Detection Rule

If a child repo contains any garage-exclusive artifact:

```
audit_rules.json RULE-009: no_governance_in_car
Classification: FAIL_SCOPE
Severity: BLOCKING
Certification: FAIL
Merge: BLOCKED
```

### Enforcement Mechanism

1. Auditor evaluates RULE-009 on every work packet execution.
2. Child repo CI validates via `.garage/certification.json` presence and signature.
3. No merge without valid certification.
4. No certification without RULE-009 passing.

## 4. Agent Ownership

All agent definitions live in the Garage:

| Agent | Location | Role |
|-------|----------|------|
| Planner | `factory/agents/agent-planner/SKILL.md` | Generates WORK_PACKET from user intent |
| Builder | `factory/agents/agent-builder/SKILL.md` | Executes approved WORK_PACKET (formerly Worker) |
| Auditor | `factory/agents/agent-auditor/SKILL.md` | Evaluates execution compliance |
| DB Agent | `factory/agents/agent-db/SKILL.md` | Database governance specialist |

Agents are invoked by the Garage orchestration pipeline. They do not self-invoke. They do not reside in child repos.

### Agent Isolation

- No agent holds multiple roles simultaneously.
- No peer-to-peer agent communication.
- Artifacts are the only handoff mechanism.
- Human is sovereign at every decision point.

## 5. Certification as Merge Gate

No child repo PR merges without a valid Garage certification.

### Certification Requirements

| Requirement | Check |
|-------------|-------|
| certification.json exists | `.garage/certification.json` present in PR branch |
| Schema valid | Validates against `factory/contracts/certification.schema.json` |
| doctrine_version current | Matches `factory/contracts/doctrine_version.json` |
| audit_status = PASS | Derived from auditor classification |
| Signature valid | HMAC-SHA256 validates against GARAGE_SIGNING_KEY |
| Artifact hash valid | SHA-256 of required artifacts matches `artifact_hash` field |

All 6 checks must pass. No partial pass. Fail-closed.
