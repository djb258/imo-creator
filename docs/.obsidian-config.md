# Obsidian Configuration for Barton Doctrine

> This file documents how to set up Obsidian for optimal use with the Barton Doctrine docs.

---

## Vault Setup

### Option 1: Docs as Vault Root
Set `docs/` as your Obsidian vault root:
```
Vault Path: /path/to/repo/docs/
```

### Option 2: Symlink into Existing Vault
```bash
# Link docs folder into your vault
ln -s /path/to/repo/docs /path/to/your/vault/IMO-Creator
```

FROM "docs"

## Sync with Repo

### Auto-Sync Setup
The `global_config.yaml` defines Obsidian sync:

```yaml
tools:
  obsidian:
    vault:
      name: "IMO-Creator"
      path: "docs/"
    sync:
      auto_sync: true
      watch_paths:
        - "docs/*.md"
        - "docs/altitude/*.md"
```

### Manual Sync
If auto-sync isn't set up, manually sync after pulls:
```bash
# Copy docs to vault
cp -r docs/* /path/to/vault/IMO-Creator/
```

## Templates

### New IMO Doc Template
Save as `templates/10-imo.template.md`:

```markdown
---
title: IMO Definition - {{repo_name}}
tags: [imo, status/draft]
created: {{date}}
updated: {{date}}
status: draft
---

# IMO Definition: {{repo_name}}

## Overview
[One-sentence purpose]

## INPUT PHASE
[Sources, validators, kill switches]

## MIDDLE PHASE
[Processing logic, transformations]

## OUTPUT PHASE
[Destinations, confirmations]

## Approval
- [ ] INPUT confirmed
- [ ] MIDDLE confirmed
- [ ] OUTPUT confirmed
- [ ] **APPROVED**
```

### New Altitude Doc Template
Save as `templates/altitude.template.md`:

```markdown
---
title: {{level}} - {{repo_name}}
tags: [altitude/{{level_short}}, status/draft]
created: {{date}}
updated: {{date}}
---

# {{level_name}}

> **Altitude**: {{level}}
> **Audience**: {{audience}}

## [Content based on altitude level]
```
